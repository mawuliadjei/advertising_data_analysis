Objective¶

The objective of this exercise is to build a model that can determine if a scene has a desired feature in it. Essentially, this model will be built to classify if a given scene with all it's features and characteristics provided, contains a desired feature through inference.

To begin with, the feture that will be modeled for is: h_food_str. This variable in the dataset provided is a flag for whether food is present in a scene of the TV show episodes provided.

Feature Engineering - Time Offset Variable:¶

As we have seen earlier, not all episodes have the same length. As such, in order to properly use the time_offset variable, we have to consider normalizing it for each show. Such that rather than using the number of seconds from the start of the show, we would use the percentage of the episode that has elapsed up till any given point.

This new variable can easily be expressed by:

Time Offset / Episode Length

Preprocessing¶

Train and Test Split¶

To do this properly, we have to control for bias coming from the information concerning the tv show, and the time of each episode

This means we would have to make sure the representation of each show in the training and the test sets have the same distribution as they would in the overall population.

This also means we would have to randomize the times in which we are predicting whether there is a food column or not. This will do away with any inherent bias that may lie in the sequence of the scenes in each tv show.

To do this, we use an efficient and vectorized approach to group the data according to the TV_show and each output label. We then randomly sample 30% of the values in each group to form the test set, while the rest of the data constitutes the training set.

Modelling¶

With the insights and new features from our data exploration, and the preprocessed data, we can proceed to model our data to infer which scenes have food in them.

The modeling algorithm of choice in this case is gradient boosting.

This was chosen because:

  1. Gradient Boosting performs well when the size of the data is large
  2. It handles both linear and non-linear relationships well. This is important because due to the data size and limited time for modeling, linear models (logistic regression) are not expected to perform well.
  3. It handles multicolinearity well
  4. It inherently performs feature selection due to it's tree based structure
  5. Interpretability

The Light Gradient Boosting Implementation of gradient boosting was chosen for its training efficiency without any significant loss in accuracy compared to XGBoost. It also has inbuilt means of dealing with unbalanced datasets

Model Evaluation¶

Our model evaluation will look at the following metrics.

  1. Accuracy
  2. F1 Score and Confusion Matrices
  3. Auroc

Our key metric will be the F1 Score. This is due to the imbalanced nature of the dataset. Our imbalanced dataset implies, as we will see, that we can achieve a very high accuracy score by just predicting that there is no scene with food in it in the dataset. This makes the F1 score a better metric because it considers the false predictions as well as the correct predictions. Meaning that we need to correctly predict both classes to get a good F1 score.

Benchmark Metrics¶

Before evaluating our baseline models, we need to set benchmarks to help us determine whether our the metrics we see on our baseline models are good or not.

To do this, we look at what the F1 score and the accuracy score would be in the hypothetical case where we only predict that there are no scenes with food in it.

Our benchmark scores, as seen below are:

  1. Accuracy: 91.18%
  2. F1: 0.0041

This shows the relevance of using F1 score, since the accuracy metric is clearly misleading with a high accuracy score but an F1 score that is practically 0.

Custom Evaluation Functions¶

As a final step to properly evaluate the baseline models, I have built custom functions that would help us visualize and optimize our evaluation processes.

  1. plot_metrics function: This allows us to choose an optimal cutoff point for converting our label probabilities into actual labels. This is extremely important to do since it's the best way to extract the full predictive power from our models.
  2. show_auroc function: This allows us to see the AUROC score
  3. show_confusion function: This allows us to see the confusion matrix that summarizes our predictions
In [38]:
def plot_metrics(test_lab1, pred_probs):
    from sklearn.metrics import confusion_matrix as cx
    from sklearn.metrics import accuracy_score as acc
    from sklearn.metrics import f1_score as f1
    bal_ = []
    fn_ = []
    fp_ = []
    f1_ = []
    agg_ = []
    acc_ = []
    for i in np.linspace(0.1, 0.9, 100):
        pred_1 = np.where(pred_probs[:,1] > i, 1, 0)
        acc_.append(acc(test_lab1, pred_1))
        f1_.append(f1(test_lab1, pred_1))
        bal_.append(abs((cx(test_lab1, pred_1).ravel()[1]-(cx(test_lab1, pred_1).ravel()[2]))))
        agg_.append(abs((cx(test_lab1, pred_1).ravel()[1]+(cx(test_lab1, pred_1).ravel()[2]))))
        fp_.append(cx(test_lab1, pred_1).ravel()[1])
        fn_.append(cx(test_lab1, pred_1).ravel()[2])

    plt.figure(figsize=(7,7))
    plt.plot(np.linspace(0.1, 0.9, 100), acc_, 'r-', label = 'accuracy')
    plt.plot(np.linspace(0.1, 0.9, 100), f1_, 'b-', label = 'F1 Score')
    plt.legend(loc='upper left')
    plt.show()
    plt.figure(figsize=(7,7))
    plt.plot(np.linspace(0.1, 0.9, 100), bal_, 'g-', label = 'Balance')
    plt.plot(np.linspace(0.1, 0.9, 100), agg_, 'y-', label = 'Aggregate of False Predictions')
    plt.plot(np.linspace(0.1, 0.9, 100), fn_, 'r-', label = 'False Negatives')
    plt.plot(np.linspace(0.1, 0.9, 100), fp_, 'b-', label = 'False Postives')
    plt.legend(loc='upper left')
    plt.show()

    print('The highest F1 score;', max(f1_), 'occurs at a cut off probability of:',\
          np.linspace(0.1, 0.9, 100)[f1_.index(max(f1_))])
    print('The best balance;', min(bal_), 'occurs at a cut off probability of:',\
          np.linspace(0.1, 0.9, 100)[bal_.index(min(bal_))])

def show_auroc(actual_outcome, predicted_outcome):
    from sklearn import metrics
    fpr, tpr, thresholds = metrics.roc_curve(actual_outcome, predicted_outcome)
    return metrics.auc(fpr, tpr)

def show_confusion(actual_outcome, predicted_outcome):
    from sklearn.metrics import confusion_matrix as cx
    return cx(actual_outcome, predicted_outcome)

def model_function(train, test, rm_cols,param):
    model = lgb.LGBMClassifier(objective='binary',is_unbalanced = True, learning_rate=param)
    model.fit(train[train.columns.difference(rm_cols)], train.h_food_str, eval_metric='binary_logloss')

Conclusion¶

We have seen that is is possible to model the data to be able to accurately infer when a scene in a tv show has food in it or not.

From our data exploration, we learned that the progression of time through a TV episode is actually very useful in determining whether a scene has food in it or not. This shows that tv show producers follow a pattern in placing scenes with food in their shows.

The implication this has is that even without building complex statistical models, we can advise the advertiser on when to show advertising related to food. The best times as seen from the analysis is as follows:

1. First 10-15% of each TV show
2. 22% - 58% of each TV show
3. 62% - 78% of each TV show
4. Last 96% of each TV show

Moving on to more advanced modeling, we can use gradient boosting methods to model the data into a classifer in order to predict which scenes have food in them to a high degree of accuracy. We reached metrics of 99.5% accuracy and a 0.976 F1 score.

Through this model, we also learned through the feature importances that scenes that have people in them and/or are indoors have higher likelihood of helping us predict whether there is food in a scene or not. This can be seen as variables such as interior design, dress, restaurant, couch, glass, afro hairsytle are in the top 30 variables.
